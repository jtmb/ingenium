## API Patterns — Auth, Pagination, Rate Limiting, Idempotency, Conventions

### Authentication & Authorization

```text
Authorization: Bearer <token>
```

- Auth middleware runs before any handler logic
- Rate limiting runs before auth (prevent brute force)
- Auth tokens: short-lived access (15 min) + longer refresh (7 days)
- 401 for missing/invalid credentials, 403 for insufficient permissions
- Never differentiate 401/403 based on resource existence (prevents enumeration)

### Pagination — Mandatory

Every list endpoint MUST paginate.

```json
{
  "data": [...],
  "pagination": {
    "cursor": "eyJsYXN0X2lkIjogNDJ9",
    "hasMore": true,
    "total": 1543
  }
}
```

- **Cursor-based**: preferred for large datasets
- **Offset-based**: simpler for small-to-medium datasets
- **Default limit**: 20-50 items. Max limit: 100
- **Always return `hasMore` or `total`**

### Rate Limiting

```text
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 987
X-RateLimit-Reset: 1687132800
```

- Per-user and per-IP rate limiting
- Return 429 + `Retry-After` header

### Idempotency

`PUT` and `DELETE` must be idempotent. `POST` should use idempotency keys for payment/creation operations.

```text
POST /api/v1/charges
Idempotency-Key: 8f7b3c2a-9e4d-11ed-a1eb-0242ac120002
```

- Idempotency keys stored for at least 24 hours
- Keys are UUIDs generated by the client

### Request/Response Conventions

- **JSON only**: `Content-Type: application/json`
- **ISO 8601 timestamps**: `"2026-06-18T14:30:00Z"`. Always UTC.
- **Snake_case or camelCase**: pick one and enforce it everywhere
- **No envelope for single resources**, envelope for collections
- **Boolean fields**: `isActive`, `hasSubscription`
- **Empty arrays over null**: `"items": []` not `"items": null`

### HTTP Methods — Use Correctly

```text
GET     /users          — List users
GET     /users/42       — Get user 42
POST    /users          — Create a user
PUT     /users/42       — Replace user 42 (full replacement)
PATCH   /users/42       — Update user 42 (partial update)
DELETE  /users/42       — Delete user 42
```

- **Never use GET for mutations**
- **POST for creation**, return 201 + Location header
- **PUT for full replacement**, PATCH for partial updates
- **No verbs in URLs**: `POST /users` not `POST /create-user`

### Documentation

Every API needs an OpenAPI spec (`openapi.yaml` or `openapi.json`). Not optional.
- Document every endpoint, every status code, every field
- Examples for all request/response bodies
- Spec is source of truth, not the docs page
